# ABI 助手 UI 重构需求文档（PRD）

## 1. 背景与目标
当前界面为功能草稿，信息层级弱、视觉不统一、操作路径长，不满足正式交付标准。
本次重构目标是：在保留现有能力（连接、AI 对话、计划写回、预检、回滚、备份）的前提下，构建一套现代化、可扩展、可维护的桌面 UI。

## 2. 产品定位
- 产品名：ABI 智能助手
- 形态：Power BI 外部工具 + 独立桌面应用
- 核心价值：让用户以自然语言完成模型分析、变更与安全回滚

## 3. 目标用户
- BI 开发者：高频建模、度量值维护、关系排错
- 业务分析师：依赖 AI 问答与结构解读
- 团队管理员：关注变更风险、审计与回滚

## 4. 设计原则
- 清晰分区：连接、分析、执行、回滚各自独立
- 风险前置：写回必须先预检、再确认
- 任务导向：把常见路径缩短为 2-3 步
- 视觉现代化：统一色板、卡片化布局、轻量阴影、状态标签

## 5. 信息架构
采用“侧边导航 + 页面容器”架构。

页面：
1. 总览 Dashboard
- 报告连接卡
- 模型结构树
- 模型摘要卡

2. AI 对话 Chat
- 聊天记录区
- 快捷提问区
- 输入区

3. 变更执行 Writeback
- 待执行计划预览
- 动作勾选列表
- 预检与执行按钮

4. 备份回滚 Backups
- 历史快照/回滚文件列表
- 一键选择回滚执行

5. 设置 Settings
- AI 配置入口
- 权限模式切换状态

## 6. 关键交互流程
### 6.1 报告连接
- 用户点击“扫描报告” -> 选择实例 -> 点击“连接”
- 成功后刷新模型树与摘要

### 6.2 AI 生成计划
- 用户在 Chat 提问
- 若回复包含 `abi_action_plan`，自动进入 Writeback 页并生成计划

### 6.3 安全写回
- 勾选要执行的动作
- 点击“预检并执行”
- 系统显示：信息/警告/错误
- 有错误：阻止执行
- 无错误：二次确认后写回
- 写回前自动生成“快照 + 回滚脚本”

### 6.4 回滚
- 在 Backups 页选择 rollback 文件
- 系统预检 -> 确认 -> 执行

## 7. 视觉与样式规范
- 主色：蓝系（专业、稳定）
- 背景：浅灰蓝，卡片白底
- 风险色：
  - 成功：绿色
  - 警告：琥珀
  - 错误：红色
- 字体：Microsoft YaHei UI / Segoe UI
- 布局：8px 网格体系，卡片圆角，统一留白

## 8. 状态与反馈
- 顶部状态标签：当前连接、权限模式、AI 模型
- 所有关键动作必须给出状态反馈（成功/失败/阻断原因）
- 聊天日志与执行日志保持可追溯

## 9. 非功能需求
- 启动后 2 秒内完成主界面渲染
- 页面切换无闪烁
- 允许窗口缩放，最小分辨率 1280x760

## 10. 验收标准
- 可完成连接、聊天、计划识别、勾选执行、预检、写回、回滚全链路
- 预检错误能阻断执行
- 自动备份文件按日期生成
- 备份中心可浏览并回滚
- UI 视觉统一、布局稳定、中文文案一致
